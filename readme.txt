======================================================================
                  SAP Integration Suite Trace Enabler
======================================================================

본 프로그램은 SAP Integration Suite (Cloud Integration) 내에 배포된
수많은 Integration Artifact(iFlow)들의 MPL 로그 레벨(NONE, INFO, DEBUG, TRACE)을
단 한 번의 명령으로 일괄 변경하고 주기적으로 유지해주는 유틸리티입니다.

----------------------------------------------------------------------
1. 배포 구성 파일
----------------------------------------------------------------------
동료들에게 공유하거나 다른 환경에서 실행할 때는 아래의 세 개 파일을
반드시 동일한 폴더에 넣고 함께 관리해 주십시오.

  * SAP_IS_Trace_Enabler.exe : 파이썬이 설치되지 않은 환경에서도 독립 기동하는 실행 파일
  * config.json              : SAP BTP 테넌트 및 API 인증 설정 정보 파일
  * run.bat (선택 사항)        : 실행을 쉽게 돕기 위해 생성한 50분 주기 상시 실행 스크립트

----------------------------------------------------------------------
2. 설정 방법 (config.json)
----------------------------------------------------------------------
프로그램 실행 전, `config.json` 파일의 `oauth` 블록을 본인의 BTP Service Key 정보에
맞추어 수정해 주어야 합니다.

  * clientid     : BTP 서비스 키의 clientid 입력
  * clientsecret : BTP 서비스 키의 clientsecret 입력
  * url          : Cloud Integration 테넌트 주소 (https:// 및 http:// 제외, 예: your-tenant-itspaces.cfapps.ap12.hana.ondemand.com)
  * tokenurl     : OAuth 토큰 발급 주소 (예: https://your-tenant-auth.authentication.ap12.hana.ondemand.com/oauth/token)

----------------------------------------------------------------------
3. 실행 및 옵션 안내
----------------------------------------------------------------------
아래의 방법 말고도, exe 파일을 단순히 더블클릭하여 실행하여도 좋습니다.
이러한 경우에는 기본설정 (Trace level, 50분마다 반복)으로 실행됩니다.

3.1 GUI 방식
  exe 파일을 더블클릭하여 실행
3.2 CMD 방식
본 프로그램은 콘솔 기반(CLI) 프로그램이므로 PowerShell이나 명령 프롬프트(CMD)를
열고 다음 명령을 실행하여 매개변수를 직접 세팅해 사용할 수 있습니다.

  [옵션 설명]
    --level    : 설정할 로그 레벨 (NONE, INFO, DEBUG, TRACE 중 선택. 기본값: TRACE)
    --interval : 주기적 자동 갱신 간격 (분 단위, 지정 시 해당 분 단위마다 무한 기동. 기본값: 50분)
    --config   : 기본 config.json 외에 다른 설정 경로를 사용할 경우 파일 경로 지정

  [예시 명령어]
    1) 단순히 TRACE 레벨로 모든 아티팩트를 1회 일괄 변경하고 싶을 때:
       SAP_IS_Trace_Enabler.exe --level TRACE

    2) 50분마다 자동으로 아티팩트들의 로그 레벨을 TRACE로 반복 갱신하고 싶을 때:
       SAP_IS_Trace_Enabler.exe --level TRACE --interval 50

    3) 로그 레벨을 모두 INFO로 초기화하여 원복하고 싶을 때:
       SAP_IS_Trace_Enabler.exe --level INFO

----------------------------------------------------------------------
4. 주요 핵심 로직 설명
----------------------------------------------------------------------
  * 병렬 멀티스레딩 (ThreadPoolExecutor)
    기존의 순차 API 호출은 386개의 iFlow를 갱신할 때 네트워크 지연으로 인해
    약 3~4분 이상 소요되었습니다. 본 프로그램은 20개의 통신 스레드를 병렬로 운용하여
    386개 아티팩트의 설정을 단 50초 내외로 단축 완료합니다.
    스레드 독립 요청 구조를 취하고 있어 스레드 간 인증/헤더 충돌 문제가 전혀 없습니다.

  * OAuth2 인증 오토 리프레시 (Auto-refresh)
    --interval 모드 기동 시, 백그라운드에서 주기 시작 시마다 매번 새로운 토큰을 발급하여
    인증 처리를 진행하므로, 장시간 기동하더라도 세션 만료 없이 영구 기동합니다.

  * runtimeLocationId 지정
    단순 WebUI Operations Command 전송 시 발생하는 내부 누락 버그를 보완하여
    "runtimeLocationId": "cloudintegration" 타깃 바인딩 매개변수를 포함해 발송함으로써,
    API Gateway가 명령을 받아 런타임 물리 노드에 실질적으로 로그수준 변경이 저장되도록 보장합니다.

----------------------------------------------------------------------
5. 사용 시 주의 사항 및 팁 (Tip)
----------------------------------------------------------------------
  * TRACE 수준의 60분 제한
    SAP Cloud Integration 정책에 의해 TRACE 및 DEBUG 로그 수준은 설정 후
    60분이 지나면 시스템 메모리 보호를 위해 자동으로 기존 기본 레벨(INFO 등)로 원복됩니다.
    따라서 스크립트 실행 직후에 확인해야 올바르게 바뀐 것을 확인하실 수 있습니다.

  * 웹 화면 새로고침
    프로그램 수행이 완료(성공)되었음에도 모니터링 화면(Manage Integration Content 등)에서
    이전 로그 레벨 마크가 계속 노출되는 현상은 브라우저 캐싱 때문입니다.
    키보드의 [Ctrl] + [F5] 키를 같이 눌러 화면을 '강제 새로고침' 하시면 변경 상태가 정상 렌더링됩니다.

  * 윈도우 보안 경고
    안전하게 빌드된 파이썬 패키징 파일이지만, 최초 실행 시 윈도우 보안(SmartScreen) 알림창이
    나타날 수 있습니다. '추가 정보' 링크를 클릭한 후 '실행' 버튼을 선택해 시작하시면 됩니다.

  * 일시적인 네트워크 장애 복구
    대기 루프 수행 중 네트워크 단절이나 테넌트 장애로 API 호출이 멈추더라도,
    프로그램이 뻗지 않고 1분 간격으로 재시도를 지속하는 내결함성 예외 코드가 반영되어 있습니다.
